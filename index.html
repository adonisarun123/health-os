0) Executive Summary

HealthOS is a subscription SaaS for clinics and allied health—combining scheduling, automated follow-ups, patient communications, standards-based health records, voice e-prescriptions, lab referrals, marketing automation, CMS, analytics, mobile apps, and a compliance-first backbone (ABDM/ABHA, DPDP, DLT, CERT-In; HIPAA/GDPR modes for global tenants). Clinical data aligns to HL7 FHIR R4.

1) Goals, Non-Goals, KPIs

Goals (MVP, 6–9 mo)
  • ≥20% online bookings
  • Automations shipped across ≥3 specialties
  • Secure, auditable FHIR store; ABDM M1–M3
  • DLT-compliant comms (consent & quiet hours)
  • Tele-consults (HD) with eRx + summaries
Non-Goals (MVP)
  • Inpatient/IPD workflows
  • Insurance adjudication (lite claims only)
  • Direct e-pharmacy fulfillment (via partners)
KPIs
  • No-show %, utilization, wait time
  • % online vs. front-desk bookings
  • Portal/PHR activation, record shares
  • DLT success, opt-outs, review velocity
  • Tele consult CSAT, first-time fix rate

PRD-1: Scheduling, Queue & Tele-Slots

MVP
Problem & Objectives

Reduce no-shows, simplify multi-provider calendars, support telemedicine and QR-based “Scan & Share” OPD intake. Add recurring appointments & calendar sync.

User Stories
  • Book/reschedule with buffers & room/device resources
  • Recurring appointments (e.g., weekly physio x 8)
  • Tele slot with consent + secure link
  • Real-time Google/Outlook sync (read/write)
  • Scan & Share QR → token at reception
UI Flows
  1. Create Slot Type → kind, duration, buffers
  2. Book Visit → select slot → patient → visit type → confirm
  3. Set Recurrence → rule builder → exceptions → save
  4. Waitlist Fill → cancel → rank → send offer → confirm
  5. Scan & Share → QR → demographics/ABHA → token
Edge Cases
  • Double-book race → server locks
  • External calendar conflicts → show diff & resolve
  • Tele link send failure → fallback ladder
  • Late arrival → optional reflow
  • Site w/o Scan & Share → manual fallback
Acceptance Criteria
  • Drag-drop reschedule; conflict detect
  • Waitlist offer ≤ 60s post-cancel
  • Two-way calendar sync within 60s
  • Tele slot: consent + secure link + reminder
  • Scan & Share journey < 60s
KPIs
  • No-show %, utilization
  • Avg wait time
  • Rebooking rate
  • Scan & Share adoption

Data: Appointment, Schedule, Slot, Patient, RecurrenceRule, QueueToken

PRD-15: Personal Health Records (PHR)

NEW
Problem & Objectives

Give patients a centralized, secure vault for all health data with granular sharing and offline access.

User Stories
  • Upload PDF/JPEG/PNG/DICOM; OCR & auto-categorize
  • Tagging (#Cardiology), folders (“2023 Annual Checkups”), timeline view
  • End-to-end encryption; biometric login
  • Share time-bound links; role-based family access
  • FHIR/HL7 import from hospitals & labs
  • Cloud sync + offline cached mode
UI Flows
  1. Upload → OCR → suggest category/tags → confirm
  2. Create Folder/Tag → drag-drop organize → timeline updates
  3. Share → select scope + expiry → generate secure link
  4. Connect Provider → OAuth/HIP link → FHIR bundle import
Edge Cases
  • Failed OCR → fallback to manual fields
  • DICOM too large → background upload + progress
  • Expired share link → prompt reissue
Acceptance Criteria
  • OCR accuracy ≥ 90% on printed docs
  • AES-256 client-side encryption for PHR payloads
  • Complete audit logs for access/shares
KPIs
  • PHR MAU, uploads/user
  • Share link usage
  • OCR correction rate

Data: PHRDocument, Tag, ShareLink, KeyEnvelope, AuditEvent

8) Compliance Checklist → System Controls

Use this as an internal audit sheet. Add Owners, Frequency, and Evidence in your tracker.

DPDP Act (India)

Required
  • Lawful purpose & consent: Purpose-bound consent UI; consent receipts
  • Data minimization: Field catalog per flow; DSR map
  • Rights handling: Access/correction/erasure workflows with SLA
  • Security safeguards: Encryption, RBAC/ABAC, immutable audit logs
Controls
